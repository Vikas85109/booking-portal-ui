1. Project Overview

StayBooker is a modern, fully-responsive accommodation booking portal built with React. It provides a seamless user experience for browsing, searching, and booking vacation rentals and stays. The application features a clean, intuitive interface inspired by popular booking platforms like Airbnb.

Key Highlights:
  • Single Page Application (SPA) with client-side routing
  • Real-time search and filtering capabilities
  • Complete booking flow with form validation
  • Persistent wishlist and booking history
  • Fully responsive design for all devices

3. Features & Functionality

Home Page / Property Listings

  • Browse all available accommodation listings in a responsive grid layout
  • Search functionality by location, title, or property type
  • Advanced filtering options:
    • Price range (min/max)
    • Minimum rating
    • Location selection
    • Property type (Apartment, House, Villa, Condo, Cabin)
    • Number of guests
  • Stay cards displaying: images, ratings, price per night, bedrooms, bathrooms, guest capacity

Stay Details Page

  • Comprehensive property information with image gallery
  • Property specifications (bedrooms, bathrooms, maximum guests)
  • Host information with Superhost badge indicator
  • Full amenities list with icon mapping
  • Interactive map section with zoom controls
  • Review ratings breakdown by category (cleanliness, accuracy, communication, location, check-in, value)
  • Add to wishlist functionality
  • Booking widget with date selection and price calculation

Multi-Step Booking Flow

  • Step 1 - Guest Details: Name, email, phone number, arrival time, special requests
  • Step 2 - Payment Information: Credit card number, CVV, expiry date (with input masking)
  • Step 3 - Review & Confirm: Booking summary, price breakdown, confirmation
  • Real-time form validation with error messaging
  • Price calculation including service fees (12%) and cleaning fee ($45)
  • Booking confirmation with unique booking number generation

My Bookings Management

  • View upcoming and past bookings with tab navigation
  • Booking status indicators: Upcoming, Active, Completed, Cancelled
  • Days until check-in countdown display
  • Booking cancellation with refund policy (full refund within 48 hours)
  • Booking statistics dashboard (total bookings, upcoming count, total spent)

Wishlist

  • Save favorite properties for later viewing
  • Persistent storage using localStorage
  • Quick access from header navigation with badge counter
  • Remove items from wishlist

6. State Management

The application uses React Context API for global state management through BookingContext.

Context State Structure

{
  // Wishlist Management
  wishlist: [],                    // Array of stay IDs

  // Search & Filters
  searchQuery: '',                 // Current search term
  filters: {
    priceMin: '',                  // Minimum price filter
    priceMax: '',                  // Maximum price filter
    rating: '',                    // Minimum rating filter
    location: '',                  // Location filter
    propertyType: '',              // Property type filter
    guests: ''                     // Guest count filter
  },

  // Current Booking Session
  bookingData: {
    stay: null,                    // Selected stay object
    checkIn: null,                 // Check-in date
    checkOut: null,                // Check-out date
    guests: 1,                     // Number of guests
    nights: 0,                     // Calculated nights
    subtotal: 0,                   // Stay cost
    serviceFee: 0,                 // 12% service fee
    cleaningFee: 45,               // Fixed cleaning fee
    total: 0                       // Total price
  },

  // Booking History
  bookingHistory: []               // Array of confirmed bookings
}

Available Actions

Action Description
toggleWishlist(stayId) Add or remove a stay from wishlist
setSearchQuery(query) Update the search query
setFilters(filters) Update filter values
setBookingData(data) Set current booking session data
addBooking(booking) Add confirmed booking to history
cancelBooking(bookingId) Cancel an existing booking
Persistence: Wishlist and booking history are automatically saved to localStorage and restored on page reload.

7. Routing Configuration

The application uses React Router DOM v7 for client-side routing.

Route Component Description
/ Home Main listing page with search and filters
/stay/:id StayDetails Individual property details page
/booking Booking Multi-step booking form
/my-bookings MyBookings User's booking history and management
/wishlist Wishlist Saved properties collection

9. Styling Architecture

CSS Methodology

The project uses BEM (Block Element Modifier) naming convention for CSS classes.

/* Block */
.stay-card { }

/* Element */
.stay-card__image { }
.stay-card__title { }
.stay-card__price { }

/* Modifier */
.stay-card--featured { }
.stay-card__price--discounted { }

CSS Variables (Custom Properties)

:root {

10. Installation & Setup

Prerequisites

  • Node.js (v18 or higher recommended)
  • npm or yarn package manager

Installation Steps

# Clone the repository
git clone https://github.com/Vikas85109/booking-portal-ui.git

# Navigate to project directory
cd booking-portal-ui

# Install dependencies
npm install

# Start development server
npm run dev
The development server will start at http://localhost:5173 by default.

11. Available Scripts

npm run build Build production-ready bundle to /dist folder

12. Deployment Guide

Build for Production

# Create production build
npm run build

# The output will be in the /dist folder

Deployment Options

Vercel (Recommended)

  1. Push your code to GitHub
  2. Import the repository in Vercel
  3. Vercel will auto-detect Vite and configure the build
  4. Deploy with one click

Netlify

  1. Connect your GitHub repository
  2. Set build command: npm run build
  3. Set publish directory: dist
  4. Deploy

GitHub Pages

  1. Update vite.config.js with base path
  2. Build the project
  3. Deploy the /dist folder to gh-pages branch
